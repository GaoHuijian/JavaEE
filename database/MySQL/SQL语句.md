4,查询操作 
---------
* 将硬盘中标文件的内容,读取到内存中,根据需求进行特殊形式的展示
* 查询分类:
###单表查询
		- select 字段名,字段名 from 表名 where 条件
		- and or
		- in
		- like 模糊查询,针对字符串 '%'匹配任意长度的字符 '-'匹配1个字符
	+ 单表分组查询
		- group by 字段名
		* select 依次对每一个临时表进行操作处理
		- select后面出现的字段,只能读取第一行数组,
		* 例子:select sex,count(*) from student group by sex
		- having: 1,只能出现在group by后面,不能单独使用 2,对分组之后生成的临时表进行过滤
		- orde by 针对select生成的临时表
		- order by 后面出现的字段,只能来自于select指定字段
		- 顺序 asc<从小到大,默认> desc<从大到小>
		- limit : limit x 获得前x行数据  limit x,y 获得从x-1行向下移动y行
###多表查询
* 1,联接查询---->将存在一对多关系表联接,并将符合业务员条件的数据检索出来,涉及两张表的数据,并不都是可以检索出来,只有当前数据行,在另外一张表中对应关联数据时,才能被检索出来.
* 2,联合查询
* 3,子查询

####1)连接查询
* from合成的新表中,一定会有脏数据<不符合业务条件的数据,数据中主键值与外键值不相同>,因此,通过from合成的新表后,要做的第一件事,就是将合法的数据检索出来.
* 内联接查询: 只有当前数据行,在对应表中有关联数据时,才能被检索出来
* 外联接查询: 将要操作的两张表中的一张表无条件检索出来,而另一张表中的数据只有在符合条件的情况下,才能读取出来.
*  左外联接查询: from 左表  left join 右表 on 一方主键 = 多方外键
			↑:左表数据无条件,全部检索出来
			↑:右表数据只有满足 on 条件时才能被检索出来
* 右外联查询:  from 左表  right join 右表 on 一方主键 = 多方外键
			↑:右表数据无条件,全部检索出来
			↑:左表数据只有满足 on 条件时才能被检索出来

####2)联合查询
* 作用:将彼此之间没有关联关系,但结构相同的两张表合成一张表的查询.
* 结构相同:两张 ***临时表*** 的字段个数,字段类型,字段排列顺序完全相同.
* 执行过程:
	
	    select 字段1,字段2 from表1
	    union
	    select 字段1,字段2 from 表2
* 特征: 
	* 联合查询后,生成表的字段只能来自于表1的字段
	* 联合查询后,生成表的记录来自于表1和表2
	* 自动过滤掉重复的数据.
* union all 和 union

		union会过滤掉重复的数据
		union all 不会过滤掉重复的数据

###3)子查询
*   将一条查询语句嵌套在其他的查询语句
*   子查询允许出现的位置及其作用
	* 1, where  允许出现  返回具体数据,可以一个具体的数据,也可以若干数据
	* 2, select 允许出现  返回的数据作为查询中一个字段,并且可以使用select所获得的字段中数据
	* 3,from    允许出现  返回的数据作为一种临时表
	* having    语序出现  返回的具体数据,用于参与逻辑判断

5,SQL命令执行顺序
---------------
* 1, from     :将硬盘上的表复制到内存中
* 2, where    :将符合条件的数据行读出来,生成一个临时表
* 3, group by :根据字段的数据种类,将当前表分成若干临时表
* 4, having:  :过滤分组之后的临时表,将不满足条件的临时表销毁掉
* 5, select   :进行整列的数据读取,  分区select后面的字段, 每次只能读取第一行
* 6, order by  :对select获得的历史表,进行排序操作
* 6, limit     :对最终的查询结果进行截取 limit 数字:从0行开始,向下移动指定行数,limit 起始行数,向下移动行数,
